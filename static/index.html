AI LISTENER · sidecar standalone

このページは sidecar の自己テスト用. dock ウィジェットは読み込まれてるけど, /api/inject がこのサーバにはない ので注入は失敗する (dock はエラーステータスを出す). 録音 → /save → chunks/ に webm が保存されるとこまでは動作確認できる.

統合方法 · bookmarklet (推奨)

ホストのコードに触らず, 必要なときだけ dock を流し込める. 下のボタンをブックマークバーに ドラッグ&ドロップ して保存:

🎧 Install Listener Dock

photo-ai-lisp などを開いた状態で上記ブックマークをクリックすると, その場で dock が注入される. リロードで消える (永続化しない). ホストの HTML に手を入れないので, リポ間の結合ゼロ.

統合方法 · 恒久的 (非推奨)

毎回入れるのが面倒なら, ホスト HTML の </body> 直前に:

<script src="http://localhost:8173/listener-dock.js"></script>

ただしホスト repo にコミットすると「sidecar 前提」という結合が永続化する. bookmarklet 方式のほうが依存関係が clean.

アーキテクチャ

┌────────────────────────────────────────────────────────────┐
│ Host page (photo-ai-lisp @ :8090)                          │
│                                                            │
│   <iframe id="terminal"> (ghostty-web WASM)  ← Gemini 本体  │
│   + sidebar + chat-bar + ... (既存)                        │
│   +                                                        │
│   <script src="http://localhost:8173/listener-dock.js">    │
│        ↓                                                   │
│   [🎧 EAR dock]  ← 本スクリプトが追加                       │
│     REC/STOP/meter                                         │
│        ↓                                                   │
│        ↓ POST webm → http://localhost:8173/save (CORS)     │
│                           ↓                                │
│                      chunks/rec_XXXX.webm 保存             │
│                      → abs_path を返却                     │
│        ↓                                                   │
│     fetch /api/inject?text=@<abs_path>... (host 同一オリジン)│
│        ↓ 400ms gap                                         │
│     fetch /api/inject?text=\n  (2-phase submit)            │
│        ↓                                                   │
│     host が PTY に流す → Gemini が解析 → ghostty iframe 表示 │
└────────────────────────────────────────────────────────────┘

現在このページで試せること

右上の EAR dock で REC → STOP すると:

  1. webm が http://localhost:8173/save に POST され保存
  2. dock が /api/inject を叩こうとする
  3. このサーバには /api/inject がないので 404 エラーが出る (正常)
  4. 実際に流したい場合は photo-ai-lisp に上記 1 行足してそっち開く
